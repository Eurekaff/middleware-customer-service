
## 11. 答辩问题与回答要点

### 11.1 为什么选择 Redis List

Redis List 支持 `LPUSH` 和 `BRPOP`，可以实现轻量级生产者消费者队列。本项目是本地课程演示，不需要 Kafka 的高吞吐日志流和消费者组能力，也不需要 RabbitMQ 的交换机和复杂路由。

### 11.2 为什么引入 Worker

客服任务处理可能包含 AI 判断、知识库检索和回复生成。如果全部放在 HTTP 请求中处理，会阻塞用户请求。Worker 独立消费 Redis 队列，可以实现异步处理和状态流转。

### 11.3 MCP Server 的作用是什么

MCP Server 将问题分类、知识库检索、回复生成、工单创建和转人工判断封装成工具，为后续接入真实大模型和标准 MCP Client 留出扩展点。

### 11.4 Redis 和 SQLite 如何分工

Redis 负责队列和最近状态缓存，适合快速读写；SQLite 负责持久化保存完整业务数据，保证会话、消息、任务和日志最终可追踪。

### 11.5 大模型不可用怎么办

系统采用“规则兜底 + 可选大模型增强”。未配置 API Key 或模型调用失败时，系统仍可通过规则和模板完成分类、转人工判断和回复生成，保证课堂演示稳定。

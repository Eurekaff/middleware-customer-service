# 实验报告提纲

## 1. 实验目的

本实验通过设计并实现一个基于 MCP 协议与 Redis 的智能客服工作流系统，理解中间件在 Web 应用中的作用。重点掌握 Redis 队列、状态缓存、异步 Worker、MCP 工具调用中间层和 SQLite 持久化之间的协作关系。

## 2. 实验题目

《基于 MCP 协议与 Redis 的智能客服工作流系统设计与实现》

## 3. 需求分析

### 3.1 项目背景

说明智能客服场景与中间件技术的关系，以及为什么该场景适合作为课程大作业。

### 3.2 功能需求

说明会话管理、消息处理、异步任务处理、MCP 工具调用、后台演示等功能。

### 3.3 非功能需求

说明本地可运行、链路可演示、代码结构清晰、数据可追踪等要求。

### 3.4 项目边界

说明本项目不实现复杂权限系统、真实大模型调用、Kafka、Kubernetes、向量数据库等复杂内容。

## 4. 系统设计

### 4.1 总体架构

说明前端、后端、Redis、Worker、MCP Server 和 SQLite 的职责。

### 4.2 处理流程

描述用户发送消息后的完整流程：

```text
用户提问 -> 后端创建任务 -> Redis 入队 -> Worker 出队 -> MCP 工具处理 -> 数据库保存 -> Redis 更新状态 -> 前端展示回复
```

### 4.3 任务状态流转

说明 `PENDING -> PROCESSING -> SUCCESS / FAILED / TRANSFERRED` 的状态流转。

## 9. 实验结果

展示四组测试样例：

2. 售后退款问题；
3. 投诉或转人工；
4. 售前咨询。

每组样例应包含用户输入、任务状态、分类结果、知识库命中、最终回复和日志截图。

## 10. 总结与不足

### 10.1 总结

总结 Redis、Worker、MCP Server 和 SQLite 在系统中的作用，以及本项目如何体现中间件技术。

### 10.2 不足

说明当前系统仍使用规则和模板模拟 AI，不具备真实语义理解能力；任务队列没有复杂重试和死信机制；前端采用轮询而非 WebSocket 推送。

### 10.3 后续改进方向

- 接入真实大模型 API；
- 增加任务重试和死信队列；
- 使用 WebSocket 推送任务状态；
- 将知识库升级为数据库或向量检索；
- 增加更完整的工单流转。
